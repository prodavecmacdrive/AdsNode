(Empty Interface States)
This document describes what is rendered in the central page containers if there is no data. Each empty state must be accompanied by a neat neutral illustration (or icon) and explanatory text.

1. index.md (Main Page)

Administrator: Open empty folder icon. Text: "You don't have any created projects yet." Below is a prominent "Add Game" button.

Client: Waiting or closed lock icon. Text: "You currently have no active projects. As soon as the team prepares the materials, they will appear here." (The add button is hidden).

2. game-page.md (Game Page)

Administrator: Playables grid icon with a "+" sign. Text: "There are no playables in this game yet." Below is a "Create Playable" button. The global assets block below the table shows: "Global assets are not uploaded" and an active Drag & Drop zone.

Client: Text: "There are no active playables in this game yet." No buttons. The assets block shows: "The file list is empty."

3. playable-page-tab-development.md (Tab: Dev & QA)

Common for both roles: Gears or loader icon. Text: "The developer has not yet uploaded working materials for review. As soon as the first screenshots or videos appear, you will be able to leave your edits here."

4. playable-page-tab-final.md (Tab: Final Export)

Common for both roles: Packed box (archive) icon. Text: "Final builds will be available for download only after the Pre-Final stage is approved."

5. globol-chat.md (Hierarchical Chat)

Common for both roles: If there are no messages and notifications yet, muted text hangs in the lower chat zone (above the input field): "The full history of project changes and correspondence will be displayed here. Send the first message...".
