Screen #5.2 Specification: Dev & QA Tab (tab-development)
This tab is the main communication zone during the creative production process. Here, the developer shows intermediate results (animations, graphics, logic pieces), and the client leaves targeted edits.

1. Tabs Navigation Bar

Dev & QA tab is active.

Condition: Available only if the "Approve" button was clicked on the previous stage (Specification tab).

2. Working Canvas (Materials Zone)
   This is the space where the developer uploads intermediate working materials for approval.

Content types: Images (UI screenshots), Video (gameplay recordings), or even mini-HTML5 frames (e.g., to test specific physics or particles).

Client's Tools (Point-and-Comment): Hovering the cursor over any uploaded media file changes the cursor to a "Pin" icon. Clicking places a numbered marker dot and opens a micro-popup to enter the edit text.

3. Sticky Batch Bar

A sticky panel that attaches to the bottom edge of the screen (but above the chat zone) only when the user has unsubmitted (draft) pins.

Content: Text "You have [X] unsubmitted edits" and a prominent "Submit Review" button.

Logic: Until the button is pressed, the developer does not see the pins and does not receive notifications.

4. Chat Zone (Bottom Block)

Our standard hierarchical log at the very bottom of the page. System notifications automatically drop here when the client clicks "Submit Review" (e.g.: "Client left 3 edits on video 'Victory_animation.mp4'"), and regular text communication is conducted here as well.

Role Model and Permissions:

Administrator: Has rights to upload working materials (videos, screenshots, animations) into the Canvas area.

Client: Cannot upload new materials to the Canvas. Uses the "Pin" tool to leave dotted comments on top of materials uploaded by the developer, and clicks the "Submit Review" button.
