Specification Tab (tab-specifications)
This tab acts as a fixed digital contract. It displays everything the client filled out during the briefing (playable creation), and this is the screen they must approve before development starts.

1. Tabs Navigation Bar (Under breadcrumbs)

Four large horizontal tabs: Specification (active) | Dev & QA | Pre-Final | Final.

Visual accent: A bright underline on the active tab, the rest are muted. If the next stage is not yet open (e.g., Specification is not approved), the Pre-Final and Final tabs are disabled (grayed out).

2. Status Indicator and Main Action (Top Bar)

A horizontal panel across the full width of the container right under the tabs.

State 1 (Awaiting actions): Yellow background, text "Awaiting approval. Please check the specification before starting". To the right, a large "Approve Specification" button.

State 2 (Approved): Green background, text "Specification approved [Date/Time]", the button is hidden.

3. Content Grid (Specification Body)
   All space under the status bar can be divided into neat cards/blocks:

Block A: Moodboard and Description (Main focus)

Wide card occupying the main part of the screen.

Displays text descriptions of mechanics, attached reference screenshots, and video link previews.

If the Specification is not yet approved, the client has an Edit button allowing them to make changes right here.

Block B: Technical Requirements and Targets

Ad networks: A compact list or logos of networks the build will be compiled for (Facebook, Google, AppLovin, etc.).

Localization: Micro-tags of selected languages (EN, FR, DE).

Deadline: Calendar icon with a fixed date.

Block C: Local Assets

File manager in a list format. This only holds source files related specifically to this playable (not global assets for the whole game). File size and format are specified.

4. Chat Zone (Bottom Block)

Our standard hierarchical log at the very bottom of the page. System notifications automatically drop here when the client clicks "Submit Review" (e.g.: "Client left 3 edits on video 'Victory_animation.mp4'"), and regular text communication is conducted here as well.

Role Model and Permissions:

Administrator: Can edit Specification fields, add or remove references until the stage is moved to the "Approved" status.

Client: Sees the Specification in "Read-Only" mode. Has exclusive access to the prominent "Approve Specification" button.
